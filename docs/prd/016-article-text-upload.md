# PRD-016: 粘贴文本创建文章支持

## 1. 背景与目标
当前“小白阅读”仅支持通过网址（URL）采集文章。为了提升灵活性，用户希望能够直接将剪贴板中的一段纯文本、笔记或零碎资料粘贴到输入框中，并将其转化为可阅读的文章，从而利用系统的“分歧断句”、“AI 摘要”和“指读翻译”功能。

## 2. 用户场景
- **场景 A**：用户看到一篇好文，粘贴 URL，系统自动爬取（现有逻辑）。
- **场景 B**：用户从 PDF 或其他无法直接访问的 App 中复制了一段长文，粘贴到输入框，直接开始阅读。

## 3. UI/UX 详细需求
### 3.1 智能输入框 (Smart Textarea)
- **控件类型**：将原有的 `input` 升级为 `textarea`。
- **自动伸缩**：
  - 默认高度：1 行。
  - 自动增长：随着内容增加自动向下伸长。
  - 最大高度：3 行内容。超过 3 行后，高度锁定，并出现内部滚动条。
- **外观一致性**：在单行（如输入短网址）状态下，其外观、Padding、边框应与原 `input` 控件保持完全一致，避免产生视觉位移。
- **实现技术**：采用 JavaScript 的 `onInput` 事件动态修改 `scrollHeight` 赋值给 `height`，配合 CSS 的 `max-height` 限制。

### 3.2 判定逻辑
- **URL 判定**：匹配正则表达式 `^https?://`。
- **文本判定**：如果不符合 URL 格式且内容不为空，则视为纯文本。

## 4. 功能逻辑 (Flow)
### 4.1 数据流转
1. **前端提交**：用户点击“阅读”或提交表单。
2. **API 接口分流 (`/api/articles/upload`)**：
   - 如果是 URL：保持原流程。
   - 如果是文本：
     - **标题提取**：提取首行或文本前 30 个字符作为 `title`。
     - **元数据同步**：`source` 设为 "文本"，`source_url` 标识为 "raw.txt"。
     - **持久化**：将原始文本（raw body）写入 R2 存储：`articles/${userId}/${articleId}/raw.txt`。
3. **Worker 异步处理 (`processArticle`)**：
   - 识别到 `source_url === "raw.txt"` 时，跳过网络爬取逻辑。
   - 直接读取 R2 的 `raw.txt`。
   - **数字化重构**：
     - 将换行符映射为 `<p>` 标签。
     - 每一段落文本调用 `splitSentences` 进行处理，包裹 `<span class="sentence">` 标签。
   - 生成最终生成的 `content.html` 并存回 R2，供阅读器渲染。

## 5. 技术约束与备注
- **数据库兼容性**：直接复用 `articles` 表结构。
- **断句引擎**：必须复用现有的 `book-worker/src/utils/sentence.ts` 逻辑，确保交互体验一致。
- **最大输入限制**：前端初步限制（如 10 万字），防止超大型文本导致 API 负载过高。
