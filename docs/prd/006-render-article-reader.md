# PRD 006: 采集文章阅读渲染 (Phase 1)

## 1. 目标

实现已采集网页文章的完整阅读体验。由于文章内容在采集时已由 Worker 预先清理并存入 R2，此阶段重点在于直接从 R2 读取 HTML 并渲染，暂不涉及 EPUB 的实时解压。

## 2. 方案

1. **内容读取**: 
   - 前端根据 `article_id` 调用 API 或直接（通过 Worker）读取 R2 中对应路径的 `content.html`。
   - 路径规则: `articles/${userId}/${articleId}/content.html`。
2. **渲染展示**:
   - 将读取到的 HTML 注入到阅读器的核心滚动区域。
   - **Article 模式特化**: 此模式下不显示章节侧边栏。
3. **安全清洗**:
   - 依然使用 `DOMPurify` 或 `HTMLRewriter` 确保渲染的内容不包含恶意脚本。

## 3. 任务清单

- [ ] **文章内容接口**: 实现能够根据 `articleId` 返回 R2 中 HTML 内容的 API。
- [x] **阅读页 UI 框架**: 已完成，支持独立滚动区域。
- [x] **模式自适应**: 已实现根据 URL 参数自动切换为 `article` 模式并隐藏目录。
- [ ] **正文一键填充**: 实现在进入页面时自动拉取并填充正文逻辑。

## 4. 进度

- **状态**: 优先进行 (HighPriority)
- **已完成**:
  - 阅读页基础布局。
  - 自动模式切换逻辑。
- **当前瓶颈**:
  - 缺乏从 R2 直接搬运 HTML 给前端展示的 API。

## 5. 备注
完成后，用户应能顺畅阅读所有已采集成功的文章。
