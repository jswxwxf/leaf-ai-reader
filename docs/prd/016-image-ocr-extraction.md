# PRD-016: 图片文本 OCR 提取

## 1. 需求背景
在阅读包含大量图表的书籍或文章时，用户经常需要获取图片中的文字信息以供搜索、笔记或 AI 进一步处理。由于阅读器正文是通过注入 HTML 渲染的，且图片往往是作为静态资源展示，用户无法直接选取其中的文字。

## 2. 核心功能
*   **图片操作入口**：在阅读器容器渲染完成后，通过 DOM 装饰器为所有图片（`<img>`）动态注入一个悬浮的 OCR 按钮。
*   **后端识别引擎**：点击按钮后，前端调取后端定义的 OCR API 接口。
*   **结果展示**：识别结果将通过一个独立的对话框（Dialog）进行展示。

## 3. 技术方案 (初探)
*   **内容装饰 (Decoration)**：在 `@content.tsx` 中使用 `useEffect` 监听渲染完成，使用原生 JS 遍历图片节点并注入操作元素。
*   **接口实现**：
    *   **后端**：利用 **Cloudflare Workers AI** 平台。使用其 Image-to-Text (Vision) 模型（如 `@cf/meta/llama-3.2-11b-vision-instruct`）进行文字提取。
    *   **数据链路**：前端发送图片 ID 或引用 -> 后端 Worker 从 R2 读取图片二进制数据 -> 调用集成在环境中的 AI 模型 -> 返回识别文本。
    *   **前端**：通过异步请求获取结果，并存入全局 Store。
*   **非 React 注入处理**：使用事件委托或直接为注入的按钮绑定异步 Action。
