# PRD 013: 剪贴板网址智能填入 (Clipboard Magic Paste)

## 1. 目标 (Objective)
通过监控浏览器剪贴板，自动识别并协助用户填入文章网址（URL），减少“复制-点击输入框-粘贴”的繁琐步骤，提升文章采集效率。

## 2. 用户场景 (User Stories)
*   **场景 1**：用户在社交媒体或新闻网站看到一篇好文章，复制 URL 后切换回 Leaf 界面点击“添加文章”。
*   **场景 2**：用户打开了“采集文章”对话框，然后去另一个标签页复制了网址，切回 Leaf 时希望网址已经在那儿了。

## 3. 核心功能 (Features)

### 3.1 剪贴板嗅探 (Clipboard Sniffing)
*   **触发时机**：
    *   当“采集文章”对话框（Modal）打开时。
    *   当页面由后台切换回前台（`window.onfocus`）且对话框处于打开状态时。
*   **逻辑**：
    *   使用 `navigator.clipboard.readText()` 读取文本。
    *   通过正则表达式验证内容是否为合法的网址（`http://` 或 `https://` 开头）。

### 3.2 交互设计 (UX/UI)
*   **智能填充按钮**：在 URL 输入框右侧或内部显示一个“魔法棒”图标或“从剪贴板填入”的小标签。
*   **气泡提示**：检测到网址后，在输入框下方显示一个轻量级气泡：“检测到剪贴板网址：[URL]，点击填入”。
*   **权限处理**：
    *   如果浏览器未授权，在用户点击“魔法棒”时引导授权。
    *   授权后记录状态，后续可尝试更自动的预填逻辑。

## 4. 技术要求 (Technical Requirements)
*   **安全性**：仅在 HTTPS 环境下生效。
*   **兼容性**：需适配 Chrome, Safari, Edge 的剪贴板权限模型。
*   **性能**：只在相关对话框挂载时启动监听，避免无谓的 CPU 占用。
